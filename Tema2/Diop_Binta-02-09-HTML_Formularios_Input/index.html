Los formularios se usan para recoger información del usuario, la cual será validada, para luego proceder al envío de estos datos a un servicio que los procese adecuadamente

2. Escribe en un documento htm con el titulo GET/POST y con tus palabras en que consiste el paso de datos mediante GET y post y las ventajas de cada uno

  • GET. Es un comando que realiza una petición URL, enviando los datos en esta misma.
    • Ventajas
      1. Las páginas generadas con GET pueden añadirse a marcadores o favoritos del navegador y páginas de marcadores
      2. Almacenamiento en caché y mejora del rendimiento
  • POST. Es otro comando de petición URL, pero en este caso los datos se envían dentro del paquete.
    • Ventajas
      1. Al enviarse los datos en el paquete y no en la URL, por lo que no son visibles y eso lo hace más seguro y discreto
      2. No hay un límite del tamaño de datos, por lo que es la mejor opción para subir archivos grandes o formularios largos

3. Escribe en un documento htm con el titulo atributos una lista de definición que explique cada atributo.
  • accept-charset
  • autocomplete
  • enctype
  • novalidate
  • target

accept-charset
Codificación de caracteres que se utilizará para pasar los contenidos del formulario, por ejemplo ISO-8859-1.
autocomplete
Toma valores on u off. Si se activa, el navegador completa automáticamente los datos del formulario basándose en los valores para estos mismos que el usuario haya rellenado en otros formularios.
enctype
Solo válido para el método POST. Indica el formato en el que los datos del formulario son enviados.
novalidate
Con este atributo los datos que normalmente validaría HTML5, se ignoran y, por consecuente, no se validan.
target
Igual al atributo de la etiqueta <a>. Si se usa el valor _blank, el formulario se abre en otra ventana.

  1. formaction

    • Indica el método de paso de los datos (GET o POST).
